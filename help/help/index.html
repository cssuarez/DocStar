Action Menu

By dibuono · March 9, 2013

The Action Menu is a drop-down list of additional options that are available to you when viewing a document. 

An action that is currently turned on is indicated by a checkmark.  To turn off a feature, select it and the checkmark will be removed.

  • View Native – This view displays as a .pdf and gives you all the available options for that file type, such as printing,  saving,  zooming, and searching.   If already viewing the document  in this display, it switches the document to be viewed inside of  the Eclipse3 viewer.
  • Thumbnails – The default setting is Off.  To display a miniature view of all the pages of  the document, select a placement  of Right or Left.  
  • Audit Info.  – This allows you to view audit information such as Modified On, Accessed On, and Created On.
  • Show Fields – When this option is selected, a panel of the available fields displays to the left of the document view.
  • Show Hits -  This hides or shows the data for a document to give more room for viewing, if needed.
  • Fit -   This  fits the image to either the width or height of the viewing area.  Width (default) displays the document to fit within the viewable width.   Height displays the document  to fit within the viewable height.
  • Assign Workflow – This allows you to select an available workflow to the document.  Select from the Name list and press Assign.
  • Approvals – This allows you to assign a status to the document of Approve, Deny, or Request Approval.  The Approval actions will appear under the History > Approvals section on the left side of the screen.  When you hover your mouse over the history entry, the pop-out message displays the status and comments.
  • Modify Security – Select this option to change the security of a document, inbox, or folder.  You can  add/remove user(s) or group(s) or change security permissions.  You are only able to change the security of one item at a time.
  • Email – NEED CLARIFICATION ON HOW THIS WORKS.  VIDEO SAYS IT WILL OPEN OUTLOOK.  THAT DID NOT HAPPEN WHEN I TRIED IT.  Multiple email options are available.  You can email a document using a combination of attachment and/or link and with or without a password.  Once you select an email option, you will be prompted to enter a Subject and Address.
  • Print – This opens the Print dialog where you can select the destination printer and number of copies.  You can print with one document selected at a time.
  • Set Cutoff Date – NEED CLARIFICATION.  This allows you to select a cutoff date from the calendar.
  • Freeze – This allows you place a freeze on a document.  (NEED TO EXPLAIN WHAT THAT MEANS.)  You will be prompted to enter a Name and Reason for the Freeze.
  • Unfreeze – This will remove a freeze on a document.  You will be prompted to select the Name of the person for whom the document freeze will be removed.

Action Editor

By dibuono · September 26, 2012

Action Editor contains various workflow actions that can be performed.

 

 

 

 Add to Chat Task

This appends a message found in the input to the workflow history.  The message will appear within the System tab. This message cannot be altered.  Enter the message to be added to history. If special formatting or multiple inputs are desired, see the Advanced Format task.

Add to Folder Task

This adds document to one or more folders, which are specified by the Path setting combined with the input. Optionally, the folder(s) may be created if necessary. Enter the name of the folder, to which the document will be added.  If multiple values are present, then the document is added to each folder.

     Settings

  • Path – This is the parent of the folder specified by the input(s).  Thus, if Path is ‘/Customers/Residential’ and the input evaluates to ‘Jones’, the document is placed in the folder ‘Jones’ within folder ‘Residential’ within top level folder ‘Customers’. Note that forward slashes (/) must be used.
  • Must Exist – If true, the task fails unless the destination folder already exists. If false, folders are created if they don’t already exist.
  • Securiy Class – This specifies the security class for folder(s) created by this task.

Add Values Task

This adds one or more values to a multi-valued output. The current value(s) of the output are retained and additional values are added. For esample, the field ‘InvoiceNumber’ may contain the value 352 before this task is executed. Executing this task with an input that evaluates to 459 will result in InvoiceNumber having two values: 352 and 459.  Enter Any number of inputs may be specified. Each input (and each value of any multi-valued input) is added to the output’s collection of values.

Advanced Format Task 

The Format String may contain plain text plus one or more format items. The format items appear as numbers in curly braces with optional format codes, such as {0} or {1:C}. The output receives the Format String with each format item replaced by the formatted value of the corresponding input.

Any number of inputs may be specified.  Each input corresponds to one format item in the Format String: the first input is {0}, the second is {1}, etc.  There must be an input for every format item. Note: if a single, multi-valued input is specified, each value of the input is formatted independently producing a multi-valued output.

     Settings

  • Format – The Format String is best described by example: ‘Invoice: {0}, Amount: {1:C}, Received on: {2:d}’ three inputs are required. The first replaces {0} with no special formatting. The second replaces {1:C} formatted as currency. The third replaces {2:d} formatted as a short date. Formatting codes such as the ‘C’ and ‘d’ in this example can be found online.
  • NoErr – If true, any errors caused by insufficient inputs or type mismatches between an input and its format code are ignored.

Append Task

This appends the input to the current value of the output as text.

The input may be multi-valued, in which case the output is assumed to be multi-valued, and each corresponding values of each are combined.

Barcode Task

This reads barcodes from an image document. May read a specific region or the whole page; may read a single page or all pages.

     Settings

  • Region – This defines the area on he page(s) to read.  If none is specified, the whole page(s) are read.
  • Page – This defines the page number to be read.  If unspecified, all pages are read.

Between Verify Task

This verifies  that the first of three input arguments is between the second and third input argument. If no output is specified, the workflow is halted if the result is false.

Three inputs are specified; their values are compared checking if the first is between (inclusive) the second and third input. All inputs may be multi-valued, in which case they should all have the same number of values. The comparison operation is repeated on the first of each input, and then the second, etc.

     Settings

  • Type – All inputs will be interpreted as the specified type. Correctly setting the type ensures proper comparison of dates and numbers.
  • Invert – If true, i reverses the comparison, allowing the task to test if an input is not greater than another, or if one value is not between two others.
  • Fail Message – This is applicable only when no output is specified. This specifies the message displayed to the user if/when the workflow is halted due to a verification failure.

Change Workflow Task

 This ends the current workflow and start another one upon completion of the current action.  Enter the name of the Workflow.

Must Exist – If true, the task fails unless the specified Workflow exists. If false, this task is effectively skipped and the original workflow continues; this may be desirable if the input is the result of OCR or read from a Barcode, and a misread is a real possibility.

Combo Box Task

This prompts the user running the workflow to select an item from a global list or enter a value.

     Settings

  • List Name – This is the name of a global which has previously been defined.
  • User Can Add To List – If available, this is a user-entered value that is currently not in the list, which can be added to the list.

Compare Verify Task

This compares two input values according to the Comparison setting: >, <, or, =. If no output is specified, the workflow is halted if the result is false. 

Two inputs are specified; their values are compared. All inputs may be multi-valued, in which case they should all have the same number of values. The comparison operation is repeated on the first of each input, and then the second, etc.

     Settings

  • Comparison – This determines how inputs are compared:  greater than (>), less than (<), and equal to (=) compare two inputs.
  • Type – All inputs will be interpreted as the specified type. Correctly setting the type ensures proper comparison of dates and numbers.
  • Invert – If true, reverses the comparison, allowing the task to test if an input is not greater than another, or if one value is not between two others.
  • Fail Message – This is applicable only when no output is specified.  It specifies the message displayed to the user if/when the workflow is halted due to a verification failure.

Display Message Task

This displays the specified message to the user along with the prompts from any user input tasks. This message may be used to provide instructions to the user.

Enter  the message to be displayed.

The dropdown entry prompts the user running the workflow to select an item from a global list.

The List Name is the name of a global list which has previously been defined.

End Action Task

This task allows part of an action to be conditionally executed.  If the input is true, the remaining tasks in this action are skipped, and processing continues with the next action or branches.

The input may be the output of a prior CompareVerify task. Any input is interpreted as true or false. If the input is not specifically one of these values, then any non-zero number or non-empty text is considered true;  or empty is considered false.

Extract Text Task

Uses ‘IFilter’ technology to retrieve the text contained in many popular native document formats.

Find Text Task

This searches for a specific text string in the input. If found, a specified number of characters, which follow that string are output.  If not, an empty string is output. For instance, one may search for ‘Invoice:’ and take the 9 characters which follow as the output.

The input can be any text, but will often be a large amount of text such as the output of OCR or Extract Text.

      Settings

  • Find – This is the test to find, such as ‘Invoice’.
  • Take – This is the maximum number of characters to output. Fewer characters may be output if the sought text is found near the end of the input or (optionally) end of a line.
  • Multiline – If false, the  the output will cease at the end of line. If true, the output can span multiple lines.  

Format Task

This formats the input,  typically a date or numeric value, according to a specified format code.  Input can be any value(s) to be formatted.
