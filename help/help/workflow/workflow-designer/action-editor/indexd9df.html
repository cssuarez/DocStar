Action Editor
Action Editor contains various workflow actions that can be performed.

&nbsp;

&nbsp;

<img class="aligncenter size-medium wp-image-411" title="action_editor" src="http://www.docstar.com/help/wp-content/uploads/2012/09/action_editor1-620×313.png" alt="" width="620" height="313" />
<h2> </h2>
<p style="text-align: left;"><strong> Add to Chat Task</strong></p>
<p style="text-align: left;">This appends a message found in the input to the workflow history.  The message will appear within the System tab. This message cannot be altered.  Enter the message to be added to history. If special formatting or multiple inputs are desired, see the Advanced Format task.</p>
<p style="text-align: left;"><strong>Add to Folder Task</strong></p>
<p style="text-align: left;">This adds document to one or more folders, which are specified by the Path setting combined with the input. Optionally, the folder(s) may be created if necessary. Enter the name of the folder, to which the document will be added.  If multiple values are present, then the document is added to each folder.</p>
<p style="text-align: left;">     Settings</p>

<ul>
<li>
<div style="text-align: left;">Path – This is the parent of the folder specified by the input(s).  Thus, if Path is ‘/Customers/Residential’ and the input evaluates to ‘Jones’, the document is placed in the folder ‘Jones’ within folder ‘Residential’ within top level folder ‘Customers’. Note that forward slashes (/) must be used.</div></li>
<li>
<div style="text-align: left;">Must Exist – If true, the task fails unless the destination folder already exists. If false, folders are created if they don’t already exist.</div></li>
<li>
<div style="text-align: left;">Securiy Class – This specifies the security class for folder(s) created by this task.</div></li>
</ul>
<p style="text-align: left;"><strong>Add Values Task</strong></p>
<p style="text-align: left;">This adds one or more values to a multi-valued output. The current value(s) of the output are retained and additional values are added. For esample, the field ‘InvoiceNumber’ may contain the value 352 before this task is executed. Executing this task with an input that evaluates to 459 will result in InvoiceNumber having two values: 352 and 459.  Enter Any number of inputs may be specified. Each input (and each value of any multi-valued input) is added to the output’s collection of values.</p>
<p style="text-align: left;"><strong>Advanced Format Task </strong></p>
<p style="text-align: left;">The Format String may contain plain text plus one or more format items. The format items appear as numbers in curly braces with optional format codes, such as {0} or {1:C}. The output receives the Format String with each format item replaced by the formatted value of the corresponding input.</p>
<p style="text-align: left;">Any number of inputs may be specified.  Each input corresponds to one format item in the Format String: the first input is {0}, the second is {1}, etc.  There must be an input for every format item. Note: if a single, multi-valued input is specified, each value of the input is formatted independently producing a multi-valued output.</p>
<p style="text-align: left;">     Settings</p>

<ul>
<li>
<div style="text-align: left;">Format – The Format String is best described by example: ‘Invoice: {0}, Amount: {1:C}, Received on: {2:d}’ three inputs are required. The first replaces {0} with no special formatting. The second replaces {1:C} formatted as currency. The third replaces {2:d} formatted as a short date. Formatting codes such as the ‘C’ and ‘d’ in this example can be found online.</div></li>
<li>
<div style="text-align: left;">NoErr – If true, any errors caused by insufficient inputs or type mismatches between an input and its format code are ignored.</div></li>
</ul>
<p style="text-align: left;"><strong>Append Task</strong></p>
<p style="text-align: left;">This appends the input to the current value of the output as text.</p>
<p style="text-align: left;">The input may be multi-valued, in which case the output is assumed to be multi-valued, and each corresponding values of each are combined.</p>
<p style="text-align: left;"><strong>Barcode Task</strong></p>
<p style="text-align: left;">This reads barcodes from an image document. May read a specific region or the whole page; may read a single page or all pages.</p>
<p style="text-align: left;">     Settings</p>

<ul>
<li>
<div style="text-align: left;">Region – This defines the area on he page(s) to read.  If none is specified, the whole page(s) are read.</div></li>
<li>
<div style="text-align: left;">Page – This defines the page number to be read.  If unspecified, all pages are read.</div></li>
</ul>
<p style="text-align: left;"><strong>Between Verify Task</strong></p>
<p style="text-align: left;">This verifies  that the first of three input arguments is between the second and third input argument. If no output is specified, the workflow is halted if the result is false.</p>
<p style="text-align: left;">Three inputs are specified; their values are compared checking if the first is between (inclusive) the second and third input. All inputs may be multi-valued, in which case they should all have the same number of values. The comparison operation is repeated on the first of each input, and then the second, etc.</p>
<p style="text-align: left;">     Settings</p>

<ul>
<li>
<div style="text-align: left;">Type – All inputs will be interpreted as the specified type. Correctly setting the type ensures proper comparison of dates and numbers.</div></li>
<li>
<div style="text-align: left;">Invert – If true, i reverses the comparison, allowing the task to test if an input is not greater than another, or if one value is not between two others.</div></li>
<li>
<div style="text-align: left;">Fail Message – This is applicable only when no output is specified. This specifies the message displayed to the user if/when the workflow is halted due to a verification failure.</div></li>
</ul>
<p style="text-align: left;"><strong>Change Workflow Task</strong></p>
<p style="text-align: left;"> This ends the current workflow and start another one upon completion of the current action.  Enter the name of the Workflow.</p>
<p style="text-align: left;"><strong>Must Exist</strong> – If true, the task fails unless the specified Workflow exists. If false, this task is effectively skipped and the original workflow continues; this may be desirable if the input is the result of OCR or read from a Barcode, and a misread is a real possibility.</p>
<p style="text-align: left;"><strong>Combo Box Task</strong></p>
<p style="text-align: left;">This prompts the user running the workflow to select an item from a global list or enter a value.</p>
<p style="text-align: left;">     Settings</p>

<ul>
<li>
<div style="text-align: left;">List Name – This is the name of a global which has previously been defined.</div></li>
<li>
<div style="text-align: left;">User Can Add To List – If available, this is a user-entered value that is currently not in the list, which can be added to the list.</div></li>
</ul>
<p style="text-align: left;"><strong>Compare Verify Task</strong></p>
<p style="text-align: left;">This compares two input values according to the Comparison setting: &gt;, &lt;, or, =. If no output is specified, the workflow is halted if the result is false. </p>
<p style="text-align: left;">Two inputs are specified; their values are compared. All inputs may be multi-valued, in which case they should all have the same number of values. The comparison operation is repeated on the first of each input, and then the second, etc.</p>
<p style="text-align: left;">     Settings</p>

<ul>
<li>
<div style="text-align: left;">Comparison – This determines how inputs are compared:  greater than (&gt;), less than (&lt;), and equal to (=) compare two inputs.</div></li>
<li>
<div style="text-align: left;">Type – All inputs will be interpreted as the specified type. Correctly setting the type ensures proper comparison of dates and numbers.</div></li>
<li>
<div style="text-align: left;">Invert – If true, reverses the comparison, allowing the task to test if an input is not greater than another, or if one value is not between two others.</div></li>
<li>
<div style="text-align: left;">Fail Message – This is applicable only when no output is specified.  It specifies the message displayed to the user if/when the workflow is halted due to a verification failure.</div></li>
</ul>
<p style="text-align: left;"><strong>Display Message Task</strong></p>
<p style="text-align: left;">This displays the specified message to the user along with the prompts from any user input tasks. This message may be used to provide instructions to the user.</p>
<p style="text-align: left;">Enter  the message to be displayed.</p>
<p style="text-align: left;">The dropdown entry prompts the user running the workflow to select an item from a global list.</p>
<p style="text-align: left;">The List Name is the name of a global list which has previously been defined.</p>
<p style="text-align: left;"><strong>End Action Task</strong></p>
<p style="text-align: left;">This task allows part of an action to be conditionally executed.  If the input is true, the remaining tasks in this action are skipped, and processing continues with the next action or branches.</p>
<p style="text-align: left;">The input may be the output of a prior CompareVerify task. Any input is interpreted as true or false. If the input is not specifically one of these values, then any non-zero number or non-empty text is considered true;  or empty is considered false.</p>
<p style="text-align: left;"><strong>Extract Text Task</strong></p>
<p style="text-align: left;">Uses ‘IFilter’ technology to retrieve the text contained in many popular native document formats.</p>
<p style="text-align: left;"><strong>Find Text Task</strong></p>
<p style="text-align: left;">This searches for a specific text string in the input. If found, a specified number of characters, which follow that string are output.  If not, an empty string is output. For instance, one may search for ‘Invoice:’ and take the 9 characters which follow as the output.</p>
<p style="text-align: left;">The input can be any text, but will often be a large amount of text such as the output of OCR or Extract Text.</p>
<p style="text-align: left;">      Settings</p>

<ul>
<li>
<div style="text-align: left;">Find – This is the test to find, such as ‘Invoice’.</div></li>
<li>
<div style="text-align: left;">Take – This is the maximum number of characters to output. Fewer characters may be output if the sought text is found near the end of the input or (optionally) end of a line.</div></li>
<li>
<div style="text-align: left;">Multiline – If false, the  the output will cease at the end of line. If true, the output can span multiple lines.  </div></li>
</ul>
<p style="text-align: left;"><strong>Format Task</strong></p>
<p style="text-align: left;">This formats the input,  typically a date or numeric value, according to a specified format code.  Input can be any value(s) to be formatted.</p>
